# RabbitMQ

## Q：RabbitMQ是的基本原理是什么？

A：RabbitMQ是一个消息队列，实现了AMQP协议。生产者和消费者之间使用消息队列进行异步通信，实现了系统解耦，削峰填谷和异步处理。

RabbitMQ的主要组件是：

1. Producter：用于发送消息
2. Consumer：用于消费消息
3. Exchange：交换机，用于根据规则转发消息
   - Direct：通过制定路由的key直接进行转发
   - Fanout：扇出路由，转发给所有的队列
   - Topic：通过模糊匹配路由，*匹配一个词，#匹配多个词
   - Headers：根据消息的Header部分进行路由
4. Queue：消息队列，用于存储消息
5. Binding：Exchange和Queue之间的关联关系

RabbitMQ的核心机制：

1. 消息确认，ACK机制，如果消息没有ACK，会重新投递避免消息丢失。
2. 消息持久化：消息可以持久化存储，
3. 发送者确认：生产者可以确认消息是否传递到了Exchange
4. 消费者Prefetch：消费者预取，消费者可以一次获取多条消息。

## Q：RabbitMQ如何保证消息不丢失

A：可以从三个对象进行考虑：

1. 生产者发送消息后可以获取确认
2. RabbitMQ开启持久化功能，确保消息遗失
3. 消费者关闭自动ACK,手动进行ACK操作

## Q：RabbitMQ如何避免重复消费

A：每一个消息都携带ID,当遇到重复ID时忽略请求

## Q：RabbitMQ如何保证消息顺序

A：一个queue就一个consumer，在consumer中维护多个内存队列，根据业务数据关键值（例如订单ID哈希值对内存队列数取模）将消息加入到不同的内存队列中，然后多个真正负责处理消息的线程去各自对应的内存队列当中获取消息进行消费。

## Q：RabbitMQ消息堆积怎么办？

A：

出现原因：

1. 消费能力不足
2. 消费者宕机
3. 消费逻辑太慢
4. 单队列瓶颈

解决办法：

1. 增加消费数量
2. 提高消费者prefetch的数量
3. 优化消费逻辑
